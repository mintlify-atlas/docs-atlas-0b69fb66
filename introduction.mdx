## What is Praetorius?

Praetorius is a small, disciplined toolchain for composers, artists, educators, festivals, and universities to publish **score-centric works pages** with synchronized audio, PDF scores with automatic page-follow, deep links, and clean UIs that drop into any site builder via a single code block.

<Note>
The defining feature: while audio plays, the PDF viewer **automatically snaps to the correct printed page**—so evaluators stay synchronized with the score and aren't lost between time and page.
</Note>

## Why Praetorius exists

Presenting or judging score-based works on the web often creates friction:

1. The evaluator doesn't know the score intimately
2. They get lost between media time and printed page
3. Judgments drift to secondary aesthetics instead of the work itself

Praetorius restores primary evidence: **the work as score + sound, tightly synchronized**.

## What Praetorius does NOT do

Praetorius is intentionally focused. It does not:

- **Host your media** — You own and host your audio/PDF files (CDN, Google Drive, or your own server)
- **Provide analytics** — Use your site's existing analytics solution
- **Handle authentication** — No login system or access control
- **Build your entire site** — Praetorius generates embeddable components for your existing site

This keeps the tool lightweight, portable, and under your control.

## How it works

<Steps>
  <Step title="Author works data">
    Use the CLI wizard (`prae add`) to create entries in `.prae/works.json` with title, audio URL, PDF URL, cues, and optional page-follow mapping.
  </Step>
  
  <Step title="Generate assets">
    Run `prae generate` to emit paste-ready JavaScript and CSS from your data, or generate a single `--embed` snippet.
  </Step>
  
  <Step title="Integrate anywhere">
    Paste the generated code into Squarespace, WordPress, Webflow, or any static site builder—or host the `dist/` folder on your own server.
  </Step>
</Steps>
