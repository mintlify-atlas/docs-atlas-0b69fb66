# prae init

Create starter script.js, styles.css, and seed configuration files for a new Praetorius project.

## Syntax

```bash
prae init [options]
```

## Options

| Option | Description | Default |
|--------|-------------|----------|
| `-o, --out <dir>` | Output directory for starter files | `prae-out` |
| `--dry-run` | Print actions without writing files | `false` |
| `-f, --force` | Overwrite files if they exist | `false` |
| `--wizard` | Run site configuration wizard after init | `false` |

## Behavior

The `init` command creates three starter files:

1. **script.js** - Starter JavaScript with demo work (Gymnopédie)
2. **styles.css** - Base CSS with theme tokens
3. **README.txt** - Quick reference for next steps

Additionally, if `.prae/works.json` and `.prae/config.json` don't exist, they are created with default values.

## Examples

### Basic initialization

```bash
prae init
```

Creates starter files in `prae-out/` directory.

### Custom output directory

```bash
prae init -o my-project
```

### Preview without writing

```bash
prae init --dry-run
```

Shows planned file operations without creating files.

### Initialize and configure site

```bash
prae init --wizard
```

Runs the site configuration wizard after creating starter files.

### Force overwrite existing files

```bash
prae init --force
```

## Files Created

### .prae/works.json

Empty works database:

```json
{
  "version": 1,
  "works": []
}
```

### .prae/config.json

Default configuration:

```json
{
  "theme": "dark",
  "output": {
    "minify": false,
    "embed": false
  },
  "ui": {
    "skin": "console"
  },
  "site": {
    "firstName": "",
    "lastName": "",
    "listLabel": "Works List"
  }
}
```

## Notes

<Note>
- Existing files are preserved unless `--force` is specified
- `.bak` backup files are created automatically when overwriting
- The starter script includes a single demo work to verify setup
</Note>
