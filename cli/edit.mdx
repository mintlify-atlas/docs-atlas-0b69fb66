# prae edit

Modify an existing work by ID. Supports both interactive wizard mode and direct flag-based updates.

## Syntax

```bash
prae edit <id> [options]
```

## Arguments

| Argument | Description |
|----------|-------------|
| `<id>` | Numeric work ID to edit |

## Options

| Option | Description |
|--------|-------------|
| `--title <str>` | Set title directly |
| `--slug <str>` | Set slug directly |
| `--one <str>` | Set oneliner (legacy alias) |
| `--oneliner <str>` | Set oneliner directly |
| `--description <str>` | Set description (Markdown) |
| `--audio <url>` | Set audio URL (empty string clears) |
| `--pdf <url>` | Set PDF URL (empty string clears) |
| `--no-cues` | Skip cue editing in interactive mode |
| `--no-score` | Skip score/page-follow editing |

## Behavior

### Interactive Mode (default)

When no flags provided, launches wizard pre-filled with current values:

```bash
prae edit 3
```

### Direct Flag Mode

When any flag provided, updates specified fields without prompts:

```bash
prae edit 3 --title "New Title" --audio ""
```

## Examples

### Interactive edit

```bash
prae edit 1
```

```bash
? Title: String Quartet No. 2
? Slug: string-quartet-2
? Oneliner: A through-composed quartet
? Long description: ## Program Note...
? Audio URL: https://example.com/new-recording.mp3
? Score PDF URL: https://example.com/score.pdf
? Edit cues? No
? Edit page-follow (score)? No
updated #1 String Quartet No. 2
```

### Update title only

```bash
prae edit 2 --title "Piano Sonata No. 1"
```

### Clear audio URL

```bash
prae edit 3 --audio ""
```

### Update multiple fields

```bash
prae edit 1 --title "Updated Title" --oneliner "New summary" --pdf "https://example.com/score.pdf"
```

### Edit without changing cues or score

```bash
prae edit 4 --no-cues --no-score
```

## Cue Editing

In interactive mode (when `--no-cues` not specified):

```bash
? Edit cues? Yes
? Cue label: @0:00
? mm:ss or seconds: 0:00
? Add another cue? Yes
? Cue label: @3:45
? mm:ss or seconds: 3:45
? Add another cue? No
```

Replaces all existing cues.

## Score/Page-Follow Editing

In interactive mode (when `--no-score` not specified):

```bash
? Edit page-follow (score)? Yes
? Printed p.1 equals PDF page: 3
? Media offset seconds: 0
? Time (mm:ss or seconds): 0:00
? Printed page number: 1
? Add another page mapping? No
```

## Validation

Changes are validated before saving:

- **Schema compliance** - All required fields present
- **Narrative warnings** - Long oneliners or embedded newlines flagged
- **Auto-normalization** - Times converted to seconds, slugs sanitized

## Output

```bash
updated #1 String Quartet No. 2
```

## Notes

<Note>
**Backup Safety**

- Original `.prae/works.json` saved as `.prae/works.json.bak`
- Failed validations prevent save
- Use `prae undo` to revert recent changes
</Note>

<Note>
**Clearing Values**

- Pass empty string to clear optional fields: `--audio ""` or `--pdf ""`
- Cues and score blocks can only be edited in interactive mode or by editing `.prae/works.json` directly
</Note>
