title: Generating Output

The `prae generate` command transforms your `.prae/works.json` into paste-ready JavaScript and CSS files that you can deploy to any site.

## Basic Usage

<Steps>
  <Step title="Generate default output">
    ```bash
    prae generate
    ```
    Creates `dist/script.js` and `dist/styles.css` using the default `console` skin.
  </Step>

  <Step title="Choose a skin">
    ```bash
    prae generate --skin vite-breeze
    ```
    Generates output with a specific UI theme.
  </Step>

  <Step title="Custom output directory">
    ```bash
    prae generate --out public/assets
    ```
    Writes files to a custom directory instead of `dist/`.
  </Step>
</Steps>

## Command Options

### Output Directory

```bash
# Default: ./dist
prae generate --out dist

# Custom directory
prae generate --out public/praetorius

# Absolute path
prae generate --out /var/www/html/assets
```

### Skin Selection

Praetorius ships with multiple built-in skins. Choose one with `--skin`:

```bash
# Console (default) - Terminal-style debugging UI
prae generate --skin console

# Vite Breeze - Liquid glass portfolio shell with split layout
prae generate --skin vite-breeze

# Docs Reader - Spacious documentation UI with left nav
prae generate --skin docs-reader

# Cards & Tabs - Portfolio dashboard with card grid
prae generate --skin cards-tabs

# Kiosk - Touch-first tiles for demos/galleries
prae generate --skin kiosk

# Typefolio - Web-book reader with two-page spread
prae generate --skin typefolio

# TypeScatter - Type-only poster wall with scatter-to-grid layout
prae generate --skin typescatter
```

<Tip>
  You can regenerate with a different skin at any time—your data in `.prae/works.json` stays the same.
</Tip>

### Theme

```bash
# Dark theme (default)
prae generate --theme dark

# Light theme
prae generate --theme light
```

Theme tokens are injected as CSS variables:
```css
/* Dark theme */
--bg: #0f0f0f;
--fg: #fff;
--dim: #8a8a8a;
--line: rgba(255,255,255,.18);

/* Light theme */
--bg: #ffffff;
--fg: #0d0d0d;
--dim: #555;
--line: rgba(0,0,0,.18);
```

### Minification

```bash
# Minified output (smaller file size)
prae generate --minify
```

Reduces file size by removing whitespace and comments. Recommended for production.

## Generated Files

After running `prae generate`, you'll find:

### `dist/script.js`

Contains:
- Your works data as runtime JavaScript
- Audio tag management
- Page-follow configuration
- HUD/UI bindings

Example output structure:
```javascript
/** AUTO-GENERATED by praetorius generate
 * Paste into a Squarespace Code block, or host as an external JS file.
 * Source data: .prae/works.json
 */
(function(){
  var fallback = {
    "works": [...],
    "pageFollowMaps": {...},
    "config": { "theme": "dark", "site": {} }
  };
  // Runtime initialization...
})();
```

### `dist/styles.css`

Contains:
- Theme tokens (light/dark)
- Skin-specific UI styles
- Component styles for buttons, HUD, console, etc.

### `dist/index.html` (with UI skins)

For skins like `vite-breeze`, an HTML template is also generated with:
- Proper theme class on `<body>`
- Linked stylesheets
- Script includes
- Host element (`#works-console`)

## Output for Different Deployment Targets

### Static Site

```bash
prae generate --out public/assets --skin vite-breeze
```

Then reference in your HTML:
```html
<link rel="stylesheet" href="/assets/styles.css">
<section id="works-console"></section>
<script src="/assets/script.js"></script>
<script type="module" src="/assets/app.js"></script>
```

### Squarespace / Site Builder

```bash
# Generate an embed snippet (see next guide)
prae generate --embed > embed.html
```

Or host `dist/script.js` and `dist/styles.css`, then include via Code block:
```html
<link rel="stylesheet" href="https://yourcdn.com/styles.css">
<section id="works-console"></section>
<script src="https://yourcdn.com/script.js"></script>
```

## Skin Comparison

| Skin | Use Case | Layout | PDF Support |
|------|----------|--------|-------------|
| `console` | Debugging, starter | Terminal | Basic |
| `vite-breeze` | Portfolio | Split (list + preview) | Full (pane) |
| `docs-reader` | Documentation | 3-column (nav + content + outline) | Inline |
| `cards-tabs` | Dashboard | Card grid + tabs | Modal |
| `kiosk` | Exhibition | Fullscreen tiles | Overlay |
| `typefolio` | Editorial | Two-page spread | Sidebar |
| `typescatter` | Experimental | Free-form scatter | Minimal |

<Warning>
  The `docs-reader` skin skips works data seeding—it's designed for documentation sites, not work portfolios.
</Warning>

## Validation

Before generating, the CLI:
- Validates `.prae/works.json` against the schema
- Checks for duplicate IDs/slugs
- Warns about missing audio/PDF URLs
- Validates page-follow mappings

Run `prae doctor` for a detailed validation report.
